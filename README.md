# crystal.points

A solo **points hint tracker** for Archipelago Pokémon Crystal, modelled on the
[Points Hint Randomizer](https://github.com/ty-grubber/crystal-games) from
crystal-games.

Upload an Archipelago multiworld spoiler. Every key item in your Crystal slot
is grouped into one of sixteen Johto/Kanto regions and scored. Each region's
**point total** is a fuzzy hint to what's hiding there. As you find items you
click them off, and the region's remaining points drain so you can deduce where
the rest are.

The spoiler is parsed entirely in your browser; nothing is uploaded anywhere.

## Points

Each scoring key item is worth a fixed number of points by importance:

| Points | Items |
| --- | --- |
| **9** | the 16 badges |
| **7** | the 7 HMs, gear cards (Pokégear, Radio/Expn/Map/Phone Card), Squirtbottle, Card Key, S.S. Ticket, Pass |
| **5** | Secretpotion, Machine Part, Clear Bell, Rainbow Wing, Basement Key, Lost Item, Red Scale, Mystery Egg |
| **4** | Fly unlocks (one per town, when fly shuffle is on) |
| **3** | Silver Wing, GS Ball, Pokédex, Bicycle, Blue Card, Coin Case, Itemfinder, the rods, Tea |

### Seed modifiers

Some items are worth more when a seed setting turns their checks on. These are
read from the spoiler's options header and applied automatically (translated
from the original's `upgradeModifier` system); active ones are listed in the
Settings panel.

| Modifier | Bumps (+2) | Active when |
| --- | --- | --- |
| Mon Locked Checks | Pokédex, Sweet Scent | `Dexsanity`/`Dexcountsanity` > 0, or `Randomize Pokemon Requests` ≠ Off |
| Hidden Items | Bicycle | `Randomize Hidden Items` ≠ No |
| National Park Bike | Bicycle | `National Park Access` = Bicycle |
| Buena Items | Blue Card | `Shopsanity` on |
| Game Corner | Coin Case | `Shopsanity` on |

Bumps stack, so with both Hidden Items and National Park Bike the Bicycle is
worth 7. Point values can also be edited per-item in the Settings panel.

## Develop

```bash
npm install
npm run dev        # regenerates data, then starts Vite on http://127.0.0.1:8770
npm run build      # production build into dist/
npm run typecheck
```

`npm run gen` regenerates the lookup tables from the AP-Crystal apworld data.
It reads `../Archipelago-Crystal/worlds/pokemon_crystal_prerelease/data/`; set
`AP_DATA_DIR` to point elsewhere. `gen` runs automatically before `dev`/`build`.
The generated tables in `src/generated/` are committed, so a build without the
apworld checkout (e.g. CI) reuses them and `gen` no-ops.

## Branching & deploy

- **`develop`**: default working branch; do day-to-day work here.
- **`main`**: deploy branch. Pushing/merging to `main` triggers
  `.github/workflows/deploy.yml`, which builds with
  `BASE_PATH=/<repo>/` and publishes `dist/` to GitHub Pages.

## How it fits together

```
spoiler .txt ─▶ parseSpoiler ─▶ placements ─┐
                                            ├─▶ scorePlacements ─▶ regionScores ─▶ UI
options header ─▶ upgradeBonuses ───────────┘
location label ─▶ regionForLocation ─▶ region
item name ─▶ itemPoints ─▶ base points
```

- **Region mapping.** Static apworld locations hit a precise generated map
  (`src/generated/locationRegions.json`). Dynamic locations a seed invents
  (shop slots, trainer rematches, Mom's Savings) aren't in `locations.json`, so
  they fall through to a runtime resolver (`src/lib/regionResolve.ts`) that
  reads the label's first segment and parses `Route N` by number, so suffixes
  like `Champion`/`Olivine` can't mislead it.
- **Spoiler formats.** Both multiworld (`Loc (Slot): Item (Receiver)`) and
  single-player (`Loc: Item`) layouts are parsed; solo files auto-select the
  one slot.
- **Data source.** Behind a `DataSource` interface (`src/lib/source.ts`) so a
  live AP-server source can be added later without touching the UI.

## Data files

Hand-curated (`src/data/`):

- `regions.json`: the 16 hint regions.
- `areaTags.json`: apworld location area-tag to region.
- `labelKeywords.json`: fallback place-name to region for the runtime resolver.
- `itemTiers.json`: apworld item id to point value.
- `upgrades.json`: seed-modifier point bumps.
- `ignoreLabels.json`: location-label substrings to skip (e.g. Pokédex checks).

Generated by `scripts/generate-data.mjs` (committed; rebuilt by `gen`):

- `src/generated/locationRegions.json`, `src/generated/itemPoints.json`.

## Scope

Solo only for now. Host/Join/spectator multiplayer and a live AP-server data
source are out of scope for this version.
